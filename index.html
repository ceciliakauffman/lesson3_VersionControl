What even is Version Control? Highlight #1

Version control is crucial to pretty much any coding project. Whether a freelance coder or an IT genius or a student or a master developer, code versioning records the past, present, and future of a file or code. Version control keeps a full history of everything you’ve even thought of doing to your code. Maybe not that extreme, but close enough. There are many different ways to version control, but the most basic is simply to copy the different versions to one folder. While easy, it’s extremely error prone. Centralized version control on the other hand allows for a repository to be seen by many different people, and can be viewed from different devices. As different as these are, it is only two examples of the many different options you have when it comes to code versioning.

What is Git? Highlight #2

Git, while confusing, is said to have started from “creative destruction and fiery controversy,” whatever that means. The Linux kernel was the first thing that could potentially work like a Version Control system, but that didn’t last very long, 1991-2002 to be exact. Theirs was something called BitKeeper, which spawned the idea for the first code versioning system. In 2005, BitKeeper couldn’t be free anymore, so the creators saw this as an opportunity to grow. They aimed to create a new system that was faster, stronger, simpler in design, and could handle non-linear development. That is where Git was born, and as it has evolved, it has only become faster, more responsive, and still incredibly effective.

Let’s Get Down to Basics! Highlight #3

First things first, let’s review what Git even is. Git is very different from other VCS’s as it stores and thinks about data in very different ways. While most store and think about the data in a series of lists and folders, Git sees the data as much more of snapshots, thinking about our data in a way that works like a miniature filing system. Every time you commit or save your data, Git takes a snapshot to have stored for reference. Additionally and very conveiniently, Git mostly only uses local files, no need to worry about having them backed up in different servers. Git will never get rid of your data, only add to it. .

What You Need to Know! Highlight #4

There are many different aspects that go into Git, but there are a few basics that are essential to making sure that your code and all the versions of it are committed to Gits memory. The first and most important thing to do with Git is to make sure that you have a repository. There are two ways to get a repository, you either create a new one or clone one that already exists. Once again, that is extremely helpful as we can see our thought processes through our code history. The dollar sign is normally what initializes or signifies that you are going to be connecting an external repository. Then, its followed by git init, git commit, git push, and so on and so forth.

GitHub Account information! Highlight #5

GitHub is the biggest and most efficient host for Git repositories, and is the central hub for millions and billions of collaborations across the world. Because it is so big and so widely used, while it is not an integral part of Git open source, there is a very high probability that if you’re using Git, you’ll have to come in contact with GitHub. Signing up for a GitHub account is pretty standard procedures, but in my case, using C9 there were procedures that followed. I needed to connect my SSH to make sure that everything was configured properly and I would be able to push my repositories back and forth.
